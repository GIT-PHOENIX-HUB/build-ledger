Three drift patterns specific to Lane C, named in advance:

**Pattern 2 — Inventing instead of auditing.** MASTER_TODO is 1187 lines of accumulated history. The pull will be to *clean it up* — consolidate similar items, rewrite vague items in clearer language, restructure the section ordering. Each of those moves is an *invention*, not an audit. The audit posture is *preserve verbatim where the meaning is intact, mark stale where status has changed, mark uncertain where you cannot tell*. If you find yourself rewriting a task's wording for clarity, stop. The original wording is the artifact; clarity is a content change. Surface it as `OPEN_QUESTION` if the original wording is genuinely unclear.

**Pattern 3 — Treating a directory disposition proposal as a directory disposition.** The soul-context directory contains material Shane voice-dictated over time. *You are writing a proposal*, not executing a disposition. The proposal is read by Shane (with Reviewer's verdict and BBB's witness) and Shane decides. The pull will be to write the proposal as if it were the answer. The corrective is to write it as *what I see, what I propose, what I am uncertain about, what is yours to decide*.

If any section sounds like a verdict you are not authorized to make, R3 (audience-creep — writing as the Arbiter instead of the Builder) or R7 (scope creep — work-good-but-out-of-lane) applies. If any section sounds like invention rather than audit, R1 (content mismatch) applies.
