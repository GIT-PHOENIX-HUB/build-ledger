**==When The Gauntlet Technique Is Overkill (SKIP IT)==**  
**Single-file surgical edits.** If you're updating one Modelfile parameter or fixing a CSS color value, spinning up the swarm adds overhead with no return. Just open Claude Code or Codex directly and make the change. **Pure research with no build output.** When you're reading docs, absorbing information, and haven't decided what to build yet — that's a single-agent conversation, not a swarm mission. Use ChatGPT Deep Research for this (which is exactly what we're about to set up). The swarm activates AFTER research produces actionable findings. **Linear dependency chains where nothing can parallelize.** If Step B literally cannot start until Step A is completely done (like: you must verify MXFP4 before creating Modelfiles that depend on GPT-OSS), the swarm becomes a line of agents waiting on each other. Run the gate test solo, THEN fire the swarm for the parallel work after it passes. **Decision-heavy moments that need YOUR brain, not agent bandwidth.** Shane's annotated decisions (the Q&A section, the philosophy section) — those were you thinking through implications. No swarm can replace that. The swarm executes what you decide; it doesn't decide for you.
